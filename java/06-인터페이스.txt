인터페이스(java 7 기준)
	- 추상메소드와 상수 만 포함할 수 있다.
	- 역할
		- 구현클래스가 구현할 메소드의 스펙(표준)을 정의한다.
		  * 특정 인터페이스를 구현한 구현클래스들은
		    메소드 사용법이 동일하고, 인터페이스에 정의된 모든 기능이
		    구현되어 있다.
		- 구현을 담당하는 측과 사용자측 간의 표준을 정의한다.
		  * 구현을 담당하는 측과 사용자측은 각각 인터페이스을
                    기준으로 개발을 진행하면 된다.
		- 클래스들간의 결합을 느슨하게 만든다.
	- 특징
		- 하나의 클래스가 여러 인터페이스를 구현할 수 있다.
		- 인터페이스들끼리의 상속도 가능도 가능하다.
		- 인터페이스들끼리는 다중 상속도 가능하다.
		- 인터페이스 중에는 추상메소드가 하나도 없는 인터페이스도 있다.
		  * Markup-Interface라고 부른다.
		  예) 
			public interface Clonable {

			}
			public interface Serializable {

			}
		- 인터페이스도 new 키워드를 사용해서 객체 생성할 수 없다.
		- 특정 인터페이스를 구현한 객체는 인터페이스 타입의 변수로 
                  참조할 수 있다.(변수에 저장할 수 있다.)
