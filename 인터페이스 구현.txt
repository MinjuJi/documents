인터페이스 구현
	- 인터페이스 설계서
		- 인터페이스 설계서는 다른 기종 시스템 또는 컴포넌트 간 데이터 교환 및 처리를 위한 목적으로 각 시스템의 교환 데이터 및 업무, 송수신 주체 등을 포함하고 있다.
		- 작성예
			상세 기능별 인터페이스 정의서
			-----------------------------------------------------------------------------------------------------------
			인터페이스아이디	HR_INV_01				인터페이스명	급여전표발행 인터페이스

			사후조건		전표 인터페이스 후 전표발생 결과값을 반환해야 함
			파라미터		각 급여결과, 전표번호, 급여일자, 식별자

	- 내부, 외부 모듈 연계방법(EAI, ESB 연계방법)
		- EAI(Enterprise Application Integration)방식과 ESB(Enterprise Service Bus)방식은 시스템 인터페이스를 위해 외부 및 내부 모듈을 연계하는 대표적인 방법이다.
		- EAI
			- 기업에서 운용되는 서로 다른 플랫폼 및 애플리케이션들 간의 정보전달, 연계, 통합을 가능하게 해 주는 솔루션이다.
			- EAI를 사용함으로써 각 비즈니스 간 통합 및 연계성을 증대시켜 효율성을 높이고, 각 시스템 간의 확장성을 높일 수 있다.
		- ESB
			- 애플리케이션 간의 통합 측면에서 서비스 중심으로 통합을 지향하는 아키텍처 기술이다.
			- 웹 서비스 중심으로 표준화된 데이터, 버스를 통해 이 기종 애플리케이션을 유연하게 통합하는 플랫폼이다.
			- 관리 및 보안이 쉽고, 높은 수준의 품질 지원이 가능하다.
	- 인터페이스 데이터 표준
		- 인터페이스 데이터 표준은 송신자와 수신자 간에 인터페이스 되어야 할 데이터의 형식과 표준을 정의하는 것이다.

	- 인터페이스 기능 구현 정의
		- 모듈 세부 설계서
			- 모듈 세부 설계서는 하나의 독립적인 기능을 수행하는 모듈의 구성요소와 세부적인 동작을 정의한 설계서다.
			- 컴포넌트 명세서와 인터페이스 명세서가 대표적이다.

		- 인터페이스 구현
			- 데이터 통신을 통한 인터페이스
				- 애플리케이션 영역에서 인터페이스 형식에 맞춘 데이터 포맷을 인터페이스 대상으로 전송하고 이를 수신측에서 해석하는 방식이다.
				- 대표적인 데이터포맷은 XML과 JSON이다.
				- JSON(JavaScript Object Notation)
					- JSON은 속성-값 상으로 이루어진 데이터 오브젝트를 전달하기 위해 사용하는 개방형 표준 포맷이다.
					- AJAX에서 많이 사용된다.
					- XML을 대체하는 주요 데이터 포맷이다.
					- 다양한 언어에서 지원한다.
				- XML(eXtensible Markup Language)
					- 데이터 교환을 위한 다목적 마크업 언어다.
					- 다양한 종류의 데이터를 기술하는데 사용된다.
					- 다른 종류의 시스템끼리 쉽게 데이터를 주고 받을 수 있게 하는 것이 목적이다.
		- 인터페이스 보안 기능 적용
			- 시스템 모듈 간의 통신 및 정보 교환을 지원하기 때문에 데이터 변조, 탈취 및 인터페이스 모듈 자체의 보안 취약점이 있을 수 있다.
			- 대응방안
				- 시큐어 코딩
					입력 데이터 검증, API 이용, 인증/접근제어/기밀성/권한관리 적용, 에러처리, 코드품질 향상, 캡슐화
				- 암호화
					암호화 알고리즘을 사용해서 중요 데이터를 암호화한다.
